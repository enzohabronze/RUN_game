# Cart Runner Web

Вертикальный браузерный MVP для телефона и ПК: игрок с тележкой бежит по дороге, собирает полезные предметы, избегает препятствий и следит за прочностью груза.

## Запуск

Открой папку проекта и запусти локальный сервер:

```powershell
python -m http.server 5173
```

Затем открой:

```text
http://localhost:5173/cart-runner-web/
```

## Управление

- Телефон: свайп влево или вправо.
- ПК: стрелки или A/D.
- Пауза: пробел, Escape или кнопка в HUD.

## Уровни

Уровень заканчивается, когда игрок добегает до финиша маршрута. После этого игра показывает итоги: урон грузу, остаток груза, собранные предметы, очки и оценку от 1 до 3 звезд.

Звезды считаются так:

- 3 звезды: груз не ниже 85% и собрано достаточно предметов.
- 2 звезды: груз не ниже 55% и собрано около 70% нужных предметов.
- 1 звезда: игрок добежал до финиша.

Следующие уровни уже отличаются цветовой темой, длиной маршрута, скоростью, частотой генерации и новыми препятствиями.

## Замена спрайтов

Все рабочие спрайты лежат в `assets/custom` и используются как PNG. Чтобы не запутаться, заменяй существующие PNG-файлы своими картинками с теми же именами.

Пример:

```text
assets/custom/obstacles/cone.png
```

заменяешь на свой `cone.png`, и игра сразу будет использовать его.

Игрок по умолчанию берется отсюда:

```text
assets/custom/player/player.png
```

Анимации игрока есть в папках `run`, `turn-left`, `turn-right`, `hit`, но сейчас они выключены флагом `usePlayerAnimations: false` в `src/assetManifest.js`. Когда будут готовы все кадры, можно включить `true`.

Фоны:

```text
assets/custom/backgrounds/menu.png
assets/custom/backgrounds/game-over.png
assets/custom/backgrounds/levels/level-1.png
```

## Что уже есть

- Главное меню, пауза, game over.
- Экран завершения уровня с итогами, очками и звездами.
- Кнопка перехода на следующий уровень.
- Вертикальное поле под мобильный экран.
- Свайпы, клавиатура и крупные мобильные кнопки внизу.
- Генеративный поток препятствий и предметов.
- Разные препятствия: конус, ящик, яма, барьер, масло.
- Повреждение груза вместо отдельных жизней.
- Стартовый набор заменяемых спрайтов.
- Покадровая анимация игрока через manifest.
